ActionBar

Начиная с Android 3.0 (API 11), в приложениях появилась возможность использовать ActionBar, которая сочетает в себе заголовок и меню, упрощая навигацию и предоставляя быстрый доступ к частым операциям.

ActionBar заменяет Options Menu, которая была в ранних версиях Android, используя новые дополнительные параметры для меню. По умолчанию, новый проект, создаваемый в Eclipse через шаблон BlankActivity, уже содержит ActionBar.

Если запустить программу как есть, то увидим заголовок программы, а справа значок из трёх вертикальных точек (вспоминаем первый урок):

На эмуляторе в свойствах виртуального устройства следует установить кнопки Back и Home, чтобы они отображались внизу экрана. В этом случае появится значок в меню.

Если щёлкнуть по значку из трёх вертикальных точек, то увидим всплывающее меню с единственным пунктом Settings. Итак, не прикладывая никаких усилий, мы получили элемент ActionBar в своём приложении.

Давайте разбираться, что происходит за кулисами приложения. Сначала посмотрим на сгенерированный код для активности MainActivity.java:


public class MainActivity extends Activity {
 
    @Override
    public void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.main);
    }
 
    @Override
    public boolean onCreateOptionsMenu(Menu menu) {
        getMenuInflater().inflate(R.menu.main, menu);
        return true;
    }
}

Ключевой момент - код для метода onCreateOptionsMenu(). Данный метод нам уже знаком, он использовался для создания обычного меню в старых версиях Android. Так как ActionBar заменяет старое меню, то в целях совместимости код остался почти прежним. И новый код будет работать в старых устройствах, только у них не будет выводиться значок меню в заголовке и вам нужно будет использовать стандартную кнопку Menu на Android 2.3. Мы видим, что в методе идёт обращение к ресурсу меню R.menu.main. Сам ресурс находится в файле res/menu/main.xml:


<menu xmlns:android="http://schemas.android.com/apk/res/android">
    <item android:id="@+id/menu_settings"
        android:title="@string/menu_settings"
        android:orderInCategory="100"
        android:showAsAction="never" />
</menu>

В данном случае меню состоит из одного пункта. Параметры id и title не нуждаются в объяснениях. Нас интересуют два других параметра orderInCategory и showAsAction. Параметр orderInCategory позволяет задать свой порядок вывода пунктов меню. Предположим вы создали пять пунктов меню, но пока не определились с порядком их вывода на экране. Чтобы не перемещать постоянно целые блоки кода для пунктов меню в нужном порядке, можно воспользоваться данным параметром.

И, наконец, важный атрибут android:showAsAction определяет поведение меню в ActionBar. Значение never означает, что элемент меню не должен выводиться в заголовке, а только в всплывающем меню. Если вы установите значение always, то пункт Settings сразу появится в заголовке вашего приложения. Попробуйте самостоятельно. Также доступны значения ifRooms, withText и collapseActionView.

Для обработки нажатия на пункте меню нужно переопределить метод onOptionsItemSelected():


@Override
public boolean onOptionsItemSelected(MenuItem item) {
	boolean ret;
	if (item.getItemId() == R.id.action_settings) {
		// Handle Settings
		ret = true;
		Toast.makeText(this, "Зачем нажали?", Toast.LENGTH_SHORT).show();
	} else {
		ret = super.onOptionsItemSelected(item);
	}
	return ret;
}

Меняем тему для ActionBar

Когда вы создаёте приложение по умолчанию, то используется тема Holo Light with Dark Action Bar. Если вы вдруг передумали и решили использовать светлый ActionBar, то зайдите в файл res/values-v14/styles.xml и поменяйте тему <style name="AppBaseTheme" parent="android:Theme.Holo.Light.DarkActionBar"> на:


<style name="AppBaseTheme" parent="android:Theme.Holo.Light">
